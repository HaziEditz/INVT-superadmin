Atomic /adjust fix (per architect review):

One Firebase transaction on the entire passengerWallet/{key} node — bundles balance + ledger entry + metadata in a single read-modify-write that no other writer can interleave.
beforeBalanceCents captured inside the updater function from the snapshot that actually committed (not a separate pre-read).
Audit txId pre-generated and embedded into the ledger entry as auditTxId — so if the separate walletAdminAudit/{txId} write fails, we can recover the audit info from the ledger entry itself. Critical failure logged loudly.
Existence check on key-type lookups — typo'd keys now 404 instead of synthesizing a zero balance. Stops /adjust from accidentally creating wallets for non-existent passengers.

bookings.ts — every web booking now also writes passengerIndex/key/{passengerKey} = { key, createdAt } using update() so a future mobile sign-in handler can patch uid in without clobbering it.
