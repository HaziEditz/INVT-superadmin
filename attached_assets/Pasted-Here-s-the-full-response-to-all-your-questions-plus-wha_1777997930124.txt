Your specific questions:

Which Firebase path is updated when a driver is assigned?
The core job store is SQL-backed (in-memory proxy to taxitime.co.nz MSSQL). Firebase is only written for Rental jobs specifically: rentalTaxiRequests/{rentalRequestId} is PATCHed with { status: 'confirmed', assignedDriverId, assignedAt: new Date().toISOString(), jobId }.

Does the assignment write driverId, driverName, vehicleId, assignedAt?
On the job record: DriverId, VehicleId, VehicleNo, CallSign, UserFName, UserLName are set. There is no assignedAt field on the core job — it only appears in the Rental Firebase write. The driver record gets vehiclestatus = 'Picking'. If you need assignedAt on all job types, I can add it.

Do ALL booking types appear in dispatch?
Yes — the pending/assigned tabs show all non-terminal jobs regardless of service type. The accepted serviceType values are taxi, food, freight, tm. Hail and passenger-app bookings also flow through the same pipeline. Delivery jobs are the one exception — they appear in a separate DY tab, not the main queue.

When a job is completed via dispatch, does completedAt get written?
Yes, but the field is JobCompleteTime (not completedAt) — written as new Date().toISOString() across all completion paths (force-complete, driver-complete, station-complete). completedAtMs (epoch ms) is also set. The report/closed-jobs endpoint exposes this as completedAt in its response shape. If the driver app expects a field literally named completedAt, let me know and I'll add an alias.
