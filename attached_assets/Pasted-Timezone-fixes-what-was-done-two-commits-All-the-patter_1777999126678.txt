Critical driver-app field bug (flagged for the driver app team):

The completedAt field being written to completedJobs/{cid} is a local formatted string — e.g. "01/05/2026, 10:15:46 pm" — not a UTC ISO string and not a Unix timestamp. This is unparseable reliably. The correct field completedAt_ISO is being written alongside it as "2026-05-01T10:15:46.291Z" (correct UTC ISO).

The owner panel has been fixed to always prefer completedAt_ISO and only fall back to the numeric form of completedAt if that field is a number. The malformed local string is now skipped entirely. Tell the driver app team: completedAt should either be removed or stored as new Date().toISOString().

Revenue totals — owner portal vs SA portal:

The owner portal reads completedJobs/{cid} (company-scoped) for all revenue totals. The SA portal's watchdog keeps the global joback node in server memory. These are different data sources. For company 620611 specifically, joback/620611 is empty — all trips are in completedJobs/620611. The revenue fare field used is fare (primary) with fallbacks to totalFare, amount, fareAmount. Both portals should produce the same totals as long as they both read from completedJobs/{cid} — confirm the SA portal does the same.

Driver payouts: There is no separate payout calculation in the owner panel — the fare figures come directly from the fare field on each completed job record. No deduction logic is applied in the owner portal. If you need to compare payout vs gross fare (e.g. to subtract card processing fees), that information is in companySettings/{cid}/cardSettings but is not currently factored into any report totals.
